Labeling source code changes and comments

First of all, we thank you for your participation in this project.

Your task, if you choose to accept it, is to answer several questions about the consistency of changes in Java functions and their corresponding comments.

Depending on the results of this task, the best tolokers will gain access to more complicated code-labeling tasks with higher per-task payment.

Careful markup

Before starting the markup, we ask you to read the detailed meaning of all the available answers.

Checkbox Meaning
Param added/removed Parameter is introduced or removed in the code changes.
@param renamed Parameter is renamed in the code changes, and should be renamed in the comment.
@param description inconsistent @param description is no longer consistent with the param logic.
Changes don't affect param list The code changes do not affect the parameter list.
@param remains consistent The code changes affect the parameter list, but the current description remains consistent.

We expect you to be careful about the markup and will use test assignments to validate it for correctness.
