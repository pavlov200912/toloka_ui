Labeling source code changes and comments

First of all, we thank you for your participation in this project.

Your task, if you choose to accept it, is to answer several questions about Java code changes and comment in order to classify whether comment is consistent with shown code changes.

Depending on the result of this task, the best tolokers will be permitted to access more complicated tasks of annotating source code and comments.

Labeling example

The examples provided for the annotation will look like this:

Learn more about this example. You may notice that the code changes add a parameter to the function. After the change, the function takes 5 parameters. Although the commentary describes only 4 of them.

  • We have to add one @param description to the comment, therefore, to the question "Should comment be changed?" we answered "Yes"

  • Next question is "The comment must be completely overwritten?" and the answer is "No", because changes in code only concern @param part of the comment. We don't have to replace first sentence of the comment or @param part about first four params.

  • After that, we are asked "Should something be changed in @param @return or @throws?". The answer is "Yes", we want to change @param.

  • Finally, we have to choose only Param checkbox, because other options Return and Throws aren't changed in the code.
