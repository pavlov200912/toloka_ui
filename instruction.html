Labeling source code changes and comments

First of all, we thank you for your participation in this project.

Your task, if you choose to accept it, is to answer several questions about consistency of changes in Java code with corresponding comment.

Depending on the result of this task, the best tolokers will be permitted to access more complicated tasks of annotating source code and comments.

Comment Structure

We divide each JavaDoc comment into four parts: Summary, @param, @return, and @throws.

  • The Summary part usually consists of several sentences at the beginning of a comment. This part describes the general behavior of the function.

  • The @param part consists of a description of the function parameters, each parameter is described after the @param tag.

  • The @return part goes after the @return tag in the comment and describes the return value.

  • The @throws part follows after @throws or @exception tags and describes an exception that may be thrown from this method.

The comment does not necessarily contain all four parts. For example, it can only consist of @return and @param parts, this is normal.
